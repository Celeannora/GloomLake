# GloomLake Refactoring Plan

## Goal
Refactor/rebase the GloomLake MTG deck scaffold generator project, addressing technical debt and improving maintainability.

## Current State
- PySide6 GUI + CLI for Magic: The Gathering deck building
- Monolithic files (synergy_analysis.py: 1124 lines)
- Code duplication (CARD_TYPES in 6 files)
- Dual API (compute_tags in search_cards.py and mtg_utils.py)
- No card index → slow CSV scans
- Backup files accumulating

---

## Refactoring Roadmap

### Phase 1: Module Extraction (High Priority)

| Task | Details |
|------|---------|
| **1.1** Extract `synergy_analysis.py` into pass modules | Break into `pass1_tags.py`, `pass2_oracle.py`, `pass3_bridges.py`, `pass4_redundancy.py`, `pass5_dependency.py` |
| **1.2** Move GUI auto-build logic to `scripts/auto_build.py` | ~570 lines in scaffold_gui.py lines 490-1060 |
| **1.3** Consolidate `compute_tags` to single function in `mtg_utils.py` | Remove duplicate from `search_cards.py` |

### Phase 2: Data Consolidation

| Task | Details |
|------|---------|
| **2.1** Single source for `CARD_TYPES` | Keep only in `mtg_utils.py`, remove from 5 other files |
| **2.2** Create card database index | Build SQLite/index for fast lookups instead of CSV scans |
| **2.3** Add JSON export to synergy reports | Currently markdown-only |

### Phase 3: Performance & Testing

| Task | Details |
|------|---------|
| **3.1** Optimize O(n^2) 5-pass loop | Single-pass scoring in synergy_engine.py |
| **3.2** Add test coverage | Priority: synergy_analysis.py, auto_build.py |
| **3.3** Extract magic numbers to config | Composite score formula tuning mechanism |

### Phase 4: Cleanup

| Task | Details |
|------|---------|
| **4.1** Remove `scaffold_gui_backup.py` | Dead code (~1924 lines) |
| **4.2** Clean up `plans/` directory | Archive or remove unused plans |
| **4.3** Remove `scripts/scaffold_gui.py` redirect | Entry point should be root only |
